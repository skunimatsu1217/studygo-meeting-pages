検索「営業中」フィルター追加

営業中の店舗のみを絞り込めるフィルターを検索機能に追加します。
ユーザーが今すぐ利用できる店舗を素早く見つけられるようにします。

3/4会議 決定済み事項

デフォルト状態: OFF（全店舗表示）
データソース: 管理画面のドロップインカレンダー
祝日対応: 自動判定
休業中表示: 非表示でもグレーアウトでもない → 通常表示＋利用可否テキスト

独自方針: 休業中店舗の表示

休業中店舗は「非表示」でも「グレーアウト」でもなく、通常表示のまま＋利用可否の表示とする独自方針に決定。
ユーザーが休業中でも店舗情報を確認でき、営業再開時にすぐ利用できるようにする。

要検討: 「まもなく営業」表示

「まもなく営業」の表示は営業開始何分前からか？ → 「15分？」で保留中。要決定。

検索条件の拡充

検索フィルターの拡充により、ユーザーが自分に合った店舗をより見つけやすくします。
月額料金帯フィルターとソート機能を追加します。

3/4会議 決定済み事項

設備フィルター: 不要（却下）
料金フィルター: 月額のみ3段階（5千円以下/1万円以下/2万円以下）
UI形式: チップ（トグル式）
複合条件: AND結合
ソート機能: 追加する（距離/料金/評価）

決定: 設備フィルターは不要

設備フィルター（Wi-Fi/電源/個室/会議室/ドリンク/プリンター）は不要と判断。
勉強カフェの特性上、基本設備はどの店舗でも揃っているため、フィルタリングの意味が薄い。

複合条件決定: AND結合

近い店舗の自動表示

インスタベース式の駅名検索で近くの店舗を表示する機能です。
GPS許可時は現在地から、非許可時は駅名指定で、20件＋「もっと見る」形式で表示します。

3/4会議 決定済み事項

表示件数: 20件＋「もっと見る」
距離上限: なし
検索方式: インスタベース式（駅名候補から選択のみ）
ソート順: 距離順
GPS: 許可時のみ使用

インスタベース式の定義

「府中」と入力すると「府中駅」「府中本町駅」などの候補がドロップダウンで表示され、候補からの選択のみ可能（自由テキスト検索ではない）。
選択した駅の座標を基準に距離順で店舗を表示する。

検索仕様 ▼
検索方式決定: インスタベース式（駅名オートコンプリート→選択）
表示件数決定: 20件 +「もっと見る」ボタン
距離上限決定: なし（全国対象）
ソート順決定: 距離順
距離計算Haversine公式で直線距離を算出
表示情報店舗名 / 距離（km） / 営業状況バッジ / 月額最安値
GPS未許可時駅名検索（インスタベース式）にフォールバック

実装仕様 ▼
距離計算Haversine公式で直線距離を算出。
d = 2r * arcsin(sqrt(sin²((lat2-lat1)/2) + cos(lat1)*cos(lat2)*sin²((lon2-lon1)/2)))
オートコンプリート駅名マスターデータ（約9,000駅）をDB保持。前方一致で候補を最大5件表示
ページネーション初回20件 → 「もっと見る」で追加20件ずつ。offset/limitでAPI制御
GPS連携GPS許可時は自動で現在地を基準に検索。非許可時は駅名選択を求める
API GET /api/shops/nearby?lat=&lon=&offset=0&limit=20
GET /api/stations/search?q=府中&limit=5（オートコンプリート）
