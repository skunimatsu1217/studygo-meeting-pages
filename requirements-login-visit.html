ログイン簡略化（段階的情報収集）

登録時の入力項目を最小限にし、アクション時に段階的に追加情報を収集する方式です。
登録のハードルを下げて新規ユーザーの離脱を防ぎ、見学予約などの行動を促します。

決定項目の進捗 0 / 0

3/4会議 決定済み事項（6項目）

必須項目: 4項目（氏名 + メール + 電話番号 + 生年月日）
追加情報: 契約時に入力（住所・性別等）
見学予約: 4項目のみで予約可能
パスワード: 登録時に設定
クレカ登録: ドロップイン利用時に必須
プロフィール: 段階的に収集

要望: 大阪店QR → 店舗見学予約ページ

大阪店のQRコードから直接店舗の見学予約ページに遷移できるようにしたいとの要望あり。
見学WEBの店舗別URL（studygo.jp/visit/{shop-slug}）と連動させる。

SNSログイン（Google連携）

Googleアカウントでのログイン機能を追加します。
段階的に対応し、まずはGoogleのみ。LINE・Appleは将来検討。

3/4会議 決定済み事項（7項目）

対応SNS: Googleのみ（段階的）
LINE連携: 却下（将来検討）
Apple連携: 却下（将来検討）
自動連携: しない（確認ダイアログを表示）
複数SNS: 不要（1つのみ）
パスワード: SNSユーザーは不要
解除: 不可

注意: 見学予約はSNSログイン対象外

WEB見学予約ページからはSNSログインを使わず、4項目入力での予約のみとする。
SNSログインはアプリ内のログイン・新規登録画面のみで利用可能。

要検討: 解除不可の影響 / 自動連携しない場合の確認フロー

解除不可: Google連携後に解除できない仕様。メールアドレス変更時の影響は？
確認フロー: 既存メールアカウントとGoogleアカウントのメールが一致した場合、自動連携せずに確認ダイアログを表示する設計の詳細を詰める必要あり。

対応プラットフォーム仕様 ▼
Google決定: 対応（段階的導入。まずはこれのみ）
LINE却下（将来検討。MessagingAPIとの整合性確認が必要）
Apple却下（将来検討。iOSアプリではApple Signが将来必須の可能性）
複数連携決定: 不要（1アカウント1SNSのみ）
自動連携決定: しない（既存アカウントとのメール一致時は確認ダイアログ）
パスワード決定: Googleログインユーザーはパスワード不要
連携解除決定: 不可
