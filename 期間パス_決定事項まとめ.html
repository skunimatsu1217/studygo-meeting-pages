確定版 画面フロー

新規購入フロー

店舗詳細 → 期間パスタブ → パス選択 + クーポン → 開始日選択（7日以内） → 確認 + クーポン + 3DS決済 → 完了

赤枠 = 新規画面。クーポンは選択画面・確認画面の両方で入力可

延長（再購入）フロー

マイページ / 通知 → パス選択（別期間も可） → 開始日（自動：終了翌日） → 確認 + 決済 → 完了

新規購入と同じ4画面フロー。延長時もクーポン利用可

期限切れ後のチェックイン

QRスキャン → 「期限切れ」画面 → 3択提示

「期間パスを再購入」「ドロップインで利用」「月額プランに申し込む」の3択

通知フロー

残り3日前（朝9:00） → 残り1日前（朝9:00） → 期限切れ当日（朝9:00）

プッシュ通知 + HTMLメール。「延長する」→延長専用画面、「あとで」→通知消去（お知らせには残る）

ShopPlan テーブル

// 決定事項を反映した Prisma schema
model ShopPlan {
  // ... 既存フィールド ...
  planType         Int       // 1:月額, 2:ドロップイン, 3:デフォルト
                                // 4:期間パス（新規追加）

  // ↓ 期間パス専用フィールド
  durationDays     Int?      // 7, 14, 30
  // price フィールドを流用（税込価格を格納）← DB3の決定

  // 年齢割引を適用 ← DB6の決定
  // ageDiscountAge, ageDiscountRate を流用

  // 使い放題のため以下は不要（B2の決定）
  // usageLimitType, dailyMaxMinutes, totalMaxMinutes は削除
}

ShopContract テーブル

model ShopContract {
  // ... 既存フィールド ...
  paymentType       String?   // ... 既存値 ...
                                  // | "period_pass"

  // ↓ 期間パス専用フィールド
  periodPassStartDate DateTime? // 開始日（ユーザー選択）
  periodPassEndDate   DateTime? // 終了日（startDate + durationDays）
  periodPassStatus    String?   // "pending" | "active" | "expired"
                                       // cancelled は不要（B6: 返金なし）

  // B1: 同一店舗で月額+期間パス両方可のため
  // 既存の isActive とは別に期間パスの有効状態を管理
}

✅ B1の決定に伴う追加実装ポイント

「月額プランと期間パスの同時契約: 完全に自由」を選択したことで、以下の追加対応が必要です。

チェックイン判定
同一店舗で月額+期間パスの両方がある場合、期間パスを優先してチェックイン（追加課金なし）するロジックが必要
マイページ表示
月額プランカードと期間パスカードを上下に並べて表示。それぞれのステータス・操作ボタンを独立表示
契約作成API
既存の「同一店舗で契約がある場合はエラー」のバリデーションを、planType別に緩和する改修が必要
Akerun権限
月額・期間パスの両方で権限が付与されるが、期間パス終了時に権限を削除する際、月額契約の有無を確認する必要あり
契約上限
期間パスはMAXIMUM_CONTRACT(2店舗制限)に含めない
