期間パスの種類はどれにするか？

必須

提供する期間パスのラインナップを決めます。料金体系・DBの設計に直結します。

3種類セット 1週間 / 2週間 / 1ヶ月の3パターン。選びやすく運用もシンプル
2種類 2週間 / 1ヶ月のみ。短期すぎるパスは不要と判断する場合
店舗ごと自由設定 任意の日数で作成可能。柔軟だが運用が複雑になる

料金はいくらにするか？

必須

各期間パスの価格設定です。月額プラン・ドロップインとのバランスが重要です。

全店統一価格 本部が決定。例: 1週間4,378円 / 2週間7,678円 / 1ヶ月10,978円（税込）
店舗別に設定可能 立地やニーズに合わせて各加盟法人が価格設定。管理画面で設定
参考: 現行プランとの比較
  • 月額プラン: 店舗により異なる（例: 月額9,900円〜）
  • ドロップイン: 分単価 × 利用時間（上限あり）
  • 期間パスは月額より割高・ドロップインよりお得な価格帯が一般的

利用制限はつけるか？

必須

期間パスで「期間内使い放題」にするか、時間上限を設けるかを決めます。チェックイン/アウトの仕組みに影響します。

期間内使い放題 営業時間内なら回数・時間の制限なし。シンプルでわかりやすい。チェックアウト時に追加料金なし
1日の時間上限あり 例: 1日最大8時間まで。超過分は従量課金。月額プランとの差別化に有効
期間全体の時間上限あり 例: 2週間で合計50時間。利用頻度が低いユーザー向けの安価プランに

利用開始日はいつからか？

必須

購入してすぐ使えるか、開始日を選べるかで契約フローが変わります。

購入日 or 7日以内の任意日 即日利用も、旅行前の事前購入もOK。柔軟性が高い
購入日から即開始のみ シンプルだが「来週から使いたい」に対応できない
翌日開始 現行月額プランの「最短利用開始日=1日後」に合わせる

中途解約・返金はどうするか？

必須

利用開始後のキャンセルポリシーです。利用規約・Stripeの返金処理に影響します。

返金なし（買い切り） 購入後のキャンセル・返金は不可。運用がシンプルで紛争リスクも低い
開始前のみキャンセル可 利用開始日前であれば全額返金。開始後は不可
日割り返金 未使用日数分を日割りで返金。ユーザーに有利だが返金処理が複雑

購入した店舗以外でも使えるか？

推奨

月額プラン同様に「契約した店舗のみ」にするか、複数店舗で使えるようにするかです。

購入した店舗のみ 現行の月額プランと同じルール。加盟法人間の売上按分が不要でシンプル
全店舗利用可 出張・旅行ユーザーに魅力的だが、加盟法人間の売上按分が必要になる

入会金は必要か？

推奨

現行の月額プランでは入会金5,500円が必要ですが、期間パスでも徴収するかを決めます。

入会金なし お試し利用のハードルを下げる。期間パスの最大の魅力になる
入会金あり 月額プランと同様に5,500円を徴収。公平性は保てるが利用のハードルが上がる
初回のみ無料 初めて期間パスを購入する場合は入会金免除。リピート時は必要

期間パスの延長（再購入）は可能か？

推奨

期間終了後に同じパスを再購入したり、期間中に延長できるかどうかです。

延長・再購入可能 期限切れ後も再購入OK。期間中の延長は終了日翌日から新たに開始
回数制限あり 例: 連続2回まで。長期利用は月額プランへ誘導

クーポンは使えるか？

推奨

既存のクーポンシステムを期間パスにも適用するかどうかです。

専用クーポンのみ 期間パス用のクーポンを新規作成。既存クーポンとの混乱を防ぐ
既存クーポンも利用可能 現行のクーポンがそのまま使える。ただし割引率の整合性に注意
クーポン対象外 期間パスにクーポンは使えない。シンプルだが販促手段が限られる

期限通知はどのタイミング・方法で送るか？

推奨

期間パスの有効期限が近づいた際の通知タイミングと手段を決めます。ユーザー体験と延長率に直結します。

通知タイミング

3段階通知 残り3日前 / 残り1日前 / 期限切れ当日の3回。ユーザーに十分な余裕を持たせる
2段階通知 残り1日前 / 期限切れ当日の2回。通知が多すぎない
期限切れ当日のみ 最低限。実装がシンプル

通知手段

プッシュ通知 + メール アプリ通知とメールの両方。プッシュで即時、メールで確実に届ける
プッシュ通知のみ アプリ内通知のみ。メール実装不要だが、通知OFFのユーザーに届かない
メールのみ メール送信のみ。アプリ側の実装不要だがリアルタイム性に欠ける

通知内容（想定）

タイミング通知文（例）アクション
残り3日前 「期間パスの有効期限が近づいています（残り3日）」 延長ページへ誘導
残り1日前 「明日で期間パスが終了します」 延長ページへ誘導
期限切れ当日 「期間パスが終了しました。延長・再購入はこちら」 再購入ページへ誘導
購入完了時 「期間パスの購入が完了しました。○月○日〜○月○日までご利用いただけます」 利用方法の案内

自動更新オプションは将来追加するか？

後から可

期間終了時に自動で同じパスを再購入するオプション。初期リリースでは不要でも、将来拡張として検討。

契約承認制にするか？

後から可

現行月額プランでは加盟法人が「承認制」を選択可能（requiresApprovalForContract）。期間パスも同様にするかは後から設定変更可能。

期間パスの利用統計をどこまで出すか？

後から可

管理画面での統計表示（契約数、利用率、月額プランへの転換率など）。初期リリース後にデータを見ながら設計可能。

📌 期間パスとは

現行プランとの違いを整理した概要です。

一言で言うと

「1週間〜1ヶ月の固定期間、一括払い、自動終了」の新しいプランタイプ。月額のような継続課金ではなく、買い切り型。試験的にスタディGOを利用したいユーザーや短期利用者をターゲットにする。

現行プランとの比較

項目 月額プラン（現行） ドロップイン（現行） 期間パス（新規）
課金方式 月額定額・自動更新 都度従量課金 一括前払い・自動終了
契約期間 解約するまで継続 利用ごと 固定期間（1週間〜1ヶ月）
自動更新 あり（毎月15日） なし なし（期間満了で自動終了）
入会金 5,500円 不要 要検討
Stripe連携 Subscription（自動課金） PaymentIntent（都度） PaymentIntent（一括）
解約 可能（アンケート付き） 不要 要検討
ターゲット 継続利用者 一時利用者 お試し・短期利用者

ユーザーから見た契約フロー

店舗詳細 → 期間パスタブ → パス選択 → 開始日選択 → クーポン入力 → 確認・一括決済 → 完了

🏢 加盟法人への影響

加盟法人（店舗オーナー）側で確認・決定が必要な項目です。

売上・手数料

手数料率 期間パス専用の手数料率を設定するか？ 既存の commissionRate を流用するか？
支払タイミング 月額と同じ翌月精算？ 期間パスは一括払いのため即時精算も可能
全店舗利用時の按分 全店舗利用可にする場合、利用実績に応じた売上按分ロジックが必要

運用・設定

ON / OFF設定 店舗ごとに期間パスの提供を選択可能にするか？
契約承認 期間パスも承認制（requiresApprovalForContract）にするか？ 即時利用開始のため承認不要が自然
営業時間制限 shopPlanSchedule を期間パスにも適用するか？
ドア解錠 期間パスユーザーの Akerun ドア解錠権限は月額会員と同じにするか？

🗃 データベース変更案

Prisma スキーマの変更案です。決定事項が確定した後に最終化します。

ShopPlan テーブル

// Prisma schema 変更案
model ShopPlan {
  // ... 既存フィールド ...
  planType         Int       // 1:月額, 2:ドロップイン, 3:デフォルト
                                // 4:期間パス（新規追加）

  // ↓ 期間パス専用フィールド（新規追加）
  durationDays     Int?      // 有効日数（7, 14, 30 等）
  usageLimitType   String?   // "unlimited" | "daily_limit" | "total_limit"
  dailyMaxMinutes  Int?      // 1日の利用上限（分）
  totalMaxMinutes  Int?      // 期間全体の利用上限（分）
}

ShopContract テーブル

model ShopContract {
  // ... 既存フィールド ...
  paymentType      String?   // "legacy" | "hybrid" | "subscription"
                                // | "period_pass"（新規追加）

  // ↓ 期間パス専用フィールド（新規追加）
  periodPassEndDate  DateTime? // 期間パス終了日
  periodPassStatus   String?   // "active" | "expired" | "cancelled"
  totalUsedMinutes   Int?      // 累計利用時間（totalLimit使用時）
}
