1 エグゼクティブサマリー

期間パス機能の全体像を4つの観点から整理します。

WHAT - 何を作るか
固定期間の買い切りプラン
店舗が任意の日数を設定可能（7日、14日、30日など自由に設定）。期間内は使い放題で、営業時間内なら回数・時間制限なし。
WHY - なぜ必要か
月額契約のハードルが高いユーザー向けの中間オプション
ドロップインと月額の間を埋める選択肢を提供。お試し利用や短期集中利用のニーズに対応。
HOW - どう実現するか
一括前払い（Stripe PaymentIntent）
自動更新なし、期間満了で自動終了。Subscriptionを使わないシンプルな決済フロー。
IMPACT - 期待効果
新規ユーザー獲得の促進
ドロップインから月額への橋渡し。短期集中利用層の取り込み。契約ハードルの段階的な低減。

4 アプリ仕様

ユーザー向けアプリの画面設計と挙動を定義します。

タブ並び月額プラン → 期間パス → ドロップイン

4.2 購入フロー - 4画面構成 新規

1 パス選択 + クーポン入力

  • 日割り表示あり（税込価格 ÷ 日数）

  • 終了日を自動計算して表示
3 確認 + クーポン + 3DS決済
  • 料金明細（年齢割引・クーポン適用後）
  • 「キャンセル不可に同意」チェック必須

4.6 その他の画面対応

利用履歴

通常履歴に「期間パス利用」ラベル付きで混在表示。フィルタリングは不要。

領収書

発行する（既存機能に統合）。期間パス購入の領収書を一覧に追加。

お知らせ

フィルタなし（時系列で混在表示）。期間パス関連通知は既存のお知らせ一覧に統合。

店舗検索

「期間パス対応」フィルタを追加。planType=4のShopPlanが存在する店舗を抽出。

契約上限

MAXIMUM_CONTRACT（2店舗制限）に含めない。月額2店舗契約中でも期間パスは購入可能。

カード / 3DS

既存のカード登録・3Dセキュア認証フローをそのまま流用。

延長フロー

新規購入と同じ4画面フロー。別期間の選択可、クーポン利用可。

6 技術仕様

DB設計、Stripe決済、バッチ処理、Akerun連携、クーポン対応の技術詳細を定義します。

6.1 DB設計（Prismaスキーマ）

ShopPlan モデル

model ShopPlan { // 既存フィールド planType Int // 1:月額, 2:ドロップイン, 3:デフォルト, 4:期間パス(新規) name String // 流用 subTitle String? // 流用 categoryTitle String? // 流用 code String // 流用 price Int // 流用（税込価格を格納） durationDays Int? // 新規: 期間日数（店舗が任意設定） ageDiscountAge Int? // 流用: 年齢割引対象年齢 ageDiscountRate Int? // 流用: 年齢割引率 isActive Boolean // 流用 // 期間パスでは不要 // maxMinutes - 将来的に検討 // withInitialFee - 不要 // packageDuration - 不要 // packageDiscountRate - 不要 }

ShopContract モデル

model ShopContract { // 既存フィールド流用 paymentType String? // "period_pass" 追加 // 期間パス専用（新規） periodPassStartDate DateTime? // 開始日 periodPassEndDate DateTime? // 終了日 periodPassStatus String? // "pending" | "active" | "expired" // 期間パスでは不要 // stripeSubscriptionId // stripeScheduleId // subscriptionStatus // statusAsResigningUser // isConfirmedAsResigningUser }

Company モデル

model Company { // 新規追加 periodPassCommissionRate Int? // 期間パス手数料率（デフォルト=月額と同じ） }

バリデーションルール

重複チェック同一店舗で期間パスの重複契約は不可（planType別にバリデーション）
月額との共存月額 + 期間パスの同時契約は可
契約上限MAXIMUM_CONTRACT（2店舗制限）には含めない

6.2 Stripe・決済

決済方式PaymentIntent（一括払い）のみ 新規
Stripe Subscription不要（使用しない）
Webhookinvoice/subscription系イベントは不要
3Dセキュア既存実装を流用 流用

6.3 バッチ処理

期限管理バッチ（新規） - 毎日 0:00

pending → active開始日到来時にステータスを変更
active → expired終了日到来時にステータスを変更 + Akerun権限削除

通知バッチ（新規）

残り3日前プッシュ通知 + HTMLメール
残り1日前プッシュ通知 + HTMLメール
期限切れ当日プッシュ通知 + HTMLメール
送信時刻朝 9:00

月額用バッチとの関係

renewMonthlyContracts（15日サイクル）は期間パスでは不要。期間パスは独立した期限管理バッチで処理する。

6.4 Akerun連携 変更

権限付与期間連動で解錠権限を付与（開始日に付与）
権限削除期間終了時に削除（ただし月額契約の有無を確認してから）
注意事項期間パス終了時に月額契約がある場合は権限を維持

6.5 クーポン・支払い 流用

クーポン既存システムを流用（変更不要）
支払い/精算テーブル流用（reasonフィールドに期間パス用の値を追加のみ）

8 エッジケース

想定される特殊ケースとその対応方針を整理します。

ケース 対応方針
利用中に終了日到来 終了日の閉店（営業時間終了）まで利用可能。チェックイン中に0:00を跨いでも強制退出はしない。
プラン非公開時 購入済みの契約は有効のまま継続。新規購入のみ不可になる。
臨時休業時 期間は延長しない（利用規約に明記）。暦日ベースのため休業日も期間に含む。
アカウント削除時 警告表示して削除を許可（返金なし）。「期間パスの残り期間がありますが、返金はできません」旨を表示。
複数店舗同時購入 許可する。MAXIMUM_CONTRACTに含めないため、月額2店舗 + 期間パス複数の組み合わせも可。
