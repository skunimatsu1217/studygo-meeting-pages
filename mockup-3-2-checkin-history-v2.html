既存の adminShopUsages / companyShopUsages エンドポイントに planType パラメータを追加。
サーバー側で ShopPlan.planType による WHERE 条件を追加してフィルタリング。
パラメータ名 planType?: number
型 number | undefined — PLAN_TYPE定数値 (1=月額, 2=ドロップイン, 3=店舗デフォルト, 4=期間パス)
省略時 全プランタイプを返す（現在と同じ動作）

実装概要

既存の検索フォーム・テーブルにプランタイプフィルターを追加する最小変更

1. API バックエンド
definition.ts adminShopUsages / companyShopUsages の Input に
planType?: number を追加
adminShopResolver.ts adminShopUsages() の Prisma where に条件追加:
shopPlan: { planType: input.planType } (planType指定時のみ)
CompanyShopResolver.ts companyShopUsages() に同じフィルター追加
レスポンス 既存の include: { shopPlan: true } で shopPlan.planType は既に返されている → 追加不要
2. フロントエンド（Admin）
pages/admin/shopUsages/
index.tsx		 SearchParam 型に planType?: number 追加
検索フォームに ValidatableSelectBox 追加（プランタイプ選択）
プランタイプタブ UI 追加（クライアントサイド切り替え or API再取得）
pages/admin/shopUsages/
config.tsx		 テーブル列に「プランタイプ」列を追加:
key: 'planType'
formatValue: item.shopPlan.planType → バッジ表示
csvFormatValue: planType名称文字列
3. フロントエンド（Alliance）
pages/alliance/shopUsages/
index.tsx		 Admin版と同様に planType フィルター追加
pages/alliance/shopUsages/
config.tsx		 Admin版と同様にテーブル列追加
