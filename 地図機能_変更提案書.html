ホーム画面に表示される店舗データ

ホーム画面では3種類の店舗リストが横スクロールで表示されます。

① 登録住所近くの店舗

参照データ: ユーザーが登録した住所
仕組み: 登録住所をGoogle Geocoding APIで緯度経度に変換 → 近い店舗を距離順で表示
表示項目: 店舗画像、店舗名、サブ名
タップ: 店舗詳細画面へ遷移

② 近くの店舗（現在地ベース）

参照データ: スマホのGPS（現在地の緯度経度）
仕組み: 現在地からの距離を計算 → 近い順に表示
表示項目: 店舗画像、店舗名、サブ名
タップ: 店舗詳細画面へ遷移

③ お気に入り店舗

参照データ: ユーザーがお気に入り登録した店舗
表示項目: 店舗画像、店舗名、サブ名
タップ: 店舗詳細画面へ遷移
現在の問題 「登録住所近くの店舗」はGoogle Geocoding APIで住所→座標変換していますが、APIキーが未設定のため動作しない状態です。結果として「近くの店舗」（GPS）のみが機能しています。

検索画面の構成と表示データ

① 上部: 検索バー + フィルターボタン

キーワード検索で参照されるデータ:
店舗名 / サブ名 / 説明 / 市区町村 / 住所 / アクセス方法
※ 入力した文字が上記のどれかに含まれていれば該当する

② フィルター（絞り込み条件）

単一選択の条件:
利用タイプ 店舗タイプ サイズ 雰囲気 トイレ スタッフ 駅からの距離 立地タイプ
複数選択のタグ条件:
Wi-Fi 電源あり 個室 ドリンク無料
※ タグは「すべて満たす」店舗のみ表示（AND条件）

③ 中央: Google Maps 地図

地図上に表示されるもの:
マーカーは店舗タイプ別の色と赤色の2パターンがあります。

【通常マーカー】営業中 かつ 空席あり かつ 未選択:
濃い緑 = コワーキングスペース
黄緑 = 個室ブース
青緑 = 自習室
水色 = その他

【赤マーカー】以下のいずれかに該当:
🔴 ユーザーがタップして選択中（大きいサイズで表示）
🔴 営業中だが満席（利用者数 ≧ 割当座席数）
🔴 営業時間外

マーカーの判定に使うデータ:
・店舗タイプ（shopType） → マーカーの色（タイプ別4色）
・営業中かどうか → 営業時間ルールから自動計算
・満席かどうか → 割当座席数 vs 現在利用者数
・選択中かどうか → ユーザーの操作（タップ）

④ 下部: 店舗カード（横スワイプ）

カードに表示されるデータ:
・店舗画像
・店舗名 / サブ名
・アクセス方法（例: 渋谷駅 徒歩5分）
・ドロップイン料金（○○円～）
・月額料金（○○円～）
・「ルート・乗換」ボタン → Google Mapsアプリで経路案内

3.1 検索結果の並び順

距離順ソート ユーザーの現在地（GPS）または入力したエリアの中心座標からの距離を計算し、近い順に並びます。距離の計算には地球の丸みを考慮した「ハーバーサインの公式」が使われています。ただし、計算された距離（○○m）はユーザーに表示されていません。

店舗詳細画面のタブ別表示データ

① タブ「施設情報」

・アクセス方法

・利用ルール / 座席情報

画面下部（固定表示）

・「予約・チェックイン」ボタン
※ スクロール方向に応じて表示/非表示がアニメーション

地図画面で使われるデータ

① 地図の表示位置

参照データ: 店舗の緯度・経度（DBに登録済み）
表示位置: 店舗座標のやや北側にカメラを配置
ズームレベル: 16.69（建物が見える程度の拡大）
地図スタイル: コンビニ・病院等のアイコンを非表示にしたシンプルな表示

② マーカー（ピン）

参照データ:
・店舗の緯度・経度 → マーカーの位置
・店舗タイプ → マーカーの色（4種類）
・営業状態 + 座席の空き + 選択状態 → 赤に切り替わる条件

色分けルール:
店舗タイプ別の色（濃い緑/黄緑/青緑/水色）
→ 営業中・空席あり・未選択のとき表示
赤 → 以下のいずれかの場合に表示
・ユーザーが選択中（大サイズ 65px）
・満席（利用者数 ≧ 割当座席数）
・営業時間外
※ マーカーの大きさ: 通常45px / 選択中65px

③ 下部スライド（店舗情報カード）

表示データ:
・店舗画像
・店舗名 / サブ名
・住所
・営業状態バッジ

④ 経路案内

「ルート・乗換」ボタンをタップすると:
Google Mapsアプリが開き、現在地から店舗までの経路が表示される
※ URLに店舗の緯度・経度を埋め込んで起動
例: google.com/maps/dir/?api=1&destination=35.658,139.701

6. アプリとWordPressの機能比較

機能 アプリ (Flutter) WordPress連携ページ
地図で店舗を探す できる できない
マーカーで空席状況がわかる できる できない
条件で店舗を絞り込む できる できない
経路案内（ルート表示） できる できない
現在地から近い店舗を探す できる できない
店舗の詳細情報を見る できる できる
お気に入り登録 できる できない
店舗の住所→座標の自動変換 APIキー未設定 できない
まとめ アプリでは「地図で探す → 詳細を見る → 経路案内する」の流れが完成しています。しかしWordPress連携ページには地図機能が一切なく、テキストベースの店舗情報しか表示できていません。

7. データの流れ ― どこから何が参照されるか

7.1 アプリで地図を表示するまでの流れ

💾

  • 住所 / アクセス方法

  • 画像URLを生成
  • 距離を計算（GPS座標あれば）
    • →
    📱
    アプリ（Flutter）
    地図 + カードで表示
  • 緯度・経度 → マーカー位置
  • 営業中 + 利用者数 → マーカー色
  • 店舗名・料金 → カード表示
  • 緯度・経度 → 経路案内URL

    💾
    データベース
    同じ店舗マスタ
  • 緯度・経度（存在する）
  • 店舗名 / 住所
  • プラン / 営業時間

    APIサーバー

    WordPress
    地図表示なし

    8. 変更提案

    8.1 提案一覧

    # 提案内容 目的 優先度 フェーズ
    1 WordPress地図ページに地図表示を追加する WEBサイトからでも地図で店舗を探せるようにする 高 Phase 1
    2 検索結果に「距離」を表示する 「この店舗は○○m先」とユーザーに伝える 高 Phase 1
    3 Google Maps APIキーを設定する 住所→座標変換を動作させ、「登録住所近くの店舗」を機能させる 高 Phase 1
    4 WordPressページから経路案内できるようにする WEBサイトからもGoogle Mapsで経路案内を開ける 中 Phase 2
    5 地図の表示範囲で店舗を絞り込めるようにする 地図を動かしたときに「この範囲の店舗だけ」を再検索できる 中 Phase 2

    8.2 提案1: WordPress地図ページに地図表示を追加する

    現在（WordPress連携ページ）

    店舗一覧はテキストベースのみ。店舗の場所を地図で視覚的に確認することはできない。

    提案後

    Google Maps地図を表示し、アプリと同じようにマーカーで店舗の位置・営業状態を確認でき、クリックで店舗詳細に遷移できる。

    8.4 提案3: Google Maps APIキーを設定する

    何が変わるか APIキーを設定すると、ホーム画面の「お住まいの登録住所近くの店舗」セクションが正しく動作するようになります。現在はこの機能が停止しており、ユーザーの住所から近い店舗を自動的に表示できていません。
    設定箇所 何に使うか 現状
    APIサーバーの環境変数 住所→座標の変換（ホーム画面の「登録住所近くの店舗」） 未設定
    WEBフロントエンドの環境変数 ブラウザ上でのGoogle Maps地図表示 未定義
    iOSアプリの設定ファイル アプリ内のGoogle Maps地図表示 設定済み
    Androidアプリの設定ファイル アプリ内のGoogle Maps地図表示 設定済み

    8.5 提案4・5: 経路案内 & 範囲検索

    Phase 2 で対応 WordPressページからGoogle Mapsアプリを開いて経路案内できるリンクを追加します。また、地図をスクロール/ズームした際に「この範囲で再検索」ボタンで画面内の店舗だけを取得する機能を追加します。

    9. 実装スケジュール

    Phase 1（1〜2週間）：基盤整備と必須機能

    # やること 結果 日数
    1 Google Maps APIキーの取得と全環境への設定 ホーム画面の「登録住所近くの店舗」が動作するようになる 0.5日
    2 検索結果に「距離（○m）」を追加 アプリ・WEB両方で現在地からの距離が表示される 1日
    3 WordPress連携ページにGoogle Maps地図コンポーネントを追加 ブラウザ上で店舗の位置をマーカーで確認できる 2日
    4 マーカーの色分け（店舗タイプ別の色 / 満席・時間外は赤）とクリック時の情報表示 アプリと同じ見た目で店舗状態がわかる 1日

    Phase 2（1〜2週間）：機能拡張

    # やること 結果 日数
    5 WordPressページから経路案内リンクの追加 WEBサイトからGoogle Mapsで経路案内を開ける 0.5日
    6 「この範囲で検索」機能の追加 地図を動かして見える範囲だけの店舗を取得 1.5日
    7 ブラウザの現在地取得（Geolocation API）対応 WEBサイトでも現在地から近い店舗を検索できる 0.5日
    8 住所の座標変換結果をキャッシュして高速化 同じ住所の再計算を防ぎ、API料金を節約 1日
    既存のアプリへの影響 今回の提案はすべて「追加」のみです。既存のアプリの動作には一切影響しません。データベースの変更も不要で、API側は新しい項目を追加で返すだけです。
