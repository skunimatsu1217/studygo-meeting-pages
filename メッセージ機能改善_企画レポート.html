2. 現状の課題

お客様側の課題

お客様
メッセージに気づきにくい
プッシュ通知のみのため、通知を見逃すと店舗からのメッセージに長時間気づかない。アプリを開かないと新着が確認できない。
お客様
テキストのみで情報伝達に限界
画像・ファイル送信機能がないため、座席の写真や書類のやり取りができず、別途メールやLINEを使う必要がある。
お客様
リアルタイム性がない
WebSocket未実装のため、チャット画面を開いていても新着メッセージの自動更新がなく、手動で再読み込みが必要。
お客様
既読確認がわかりにくい
送ったメッセージが店舗に読まれたのかどうか、お客様側からは判別しにくい。返信を待つべきか判断できない。

店舗側の課題

店舗
未読管理が煩雑
複数ユーザーからのメッセージが一覧で確認しづらく、未返信のメッセージを見落としやすい。フィルタリング機能も不十分。
店舗
定型文がない
よくある質問への回答を毎回手入力する必要があり、返信に時間がかかる。テンプレート機能がない。
店舗
複数チャネルの管理が大変
アプリ内メッセージ、メール、LINEなど複数の連絡手段を個別に管理する必要があり、やり取りが分散してしまう。
店舗
一斉送信の操作性
お知らせやキャンペーン情報を顧客に一斉に送る際の操作が複雑。対象の絞り込みやスケジュール送信もできない。

3. 改善提案一覧

優先度：高
LINE公式アカウント連携
LINE Messaging APIを連携し、お客様がLINEからもメッセージを送受信できるように。店舗側は管理画面で一元管理。日本ユーザーの95%がLINEを利用しており、通知到達率が大幅に向上する。
優先度：高
テンプレート返信機能
店舗がよく使う返信をテンプレートとして保存し、ワンクリックで送信。営業時間・料金案内・空席状況など定型的な質問への対応を高速化。
優先度：高
未読・未返信ダッシュボード
未返信メッセージの一覧表示、経過時間のアラート、優先度の自動判定を実装。返信漏れを防止し、顧客満足度を向上。
優先度：中
画像・ファイル送信
S3を活用した画像・PDFの送受信機能。座席の写真、領収書、案内資料などを直接チャットで共有可能に。
優先度：中
クイック返信ボタン
お客様側に「空席はありますか？」「予約変更したい」などの定型質問ボタンを表示。入力の手間を減らし、問い合わせのハードルを下げる。
優先度：中
既読表示の改善
LINEのような「既読」マーク表示を導入。お客様が「読まれたかどうか」を一目で確認でき、安心感を提供。
優先度：中
一斉送信の強化
送信先のセグメント指定（契約プラン別、来店頻度別）、スケジュール送信、送信結果の分析ダッシュボードを追加。
優先度：低
自動応答Bot
営業時間外の自動返信、FAQ自動回答を実装。店舗スタッフの負担を軽減し、24時間対応を実現。

セキュリティ考慮事項

LINE Webhook署名検証
LINE Messaging APIのWebhook受信時に、X-Line-Signatureヘッダーを検証し、正規のリクエストであることを確認する。Channel Secretを使用したHMAC-SHA256署名検証を必須とする。
画像アップロードのバリデーション
アップロード可能なファイル種別（JPEG, PNG, PDF）、サイズ上限（10MB）を設定。ファイル名のサニタイズ、S3のプライベートバケットでの保存、署名付きURLでのアクセス制御を実装。
LINE User IDの安全な管理
LINE User IDは個人情報に該当するため、暗号化保存を検討。連携解除時の完全削除、アクセスログの記録を実装。
レート制限
メッセージ送信APIにレート制限を設定。LINE Messaging APIの送信上限（1,000通/分）を考慮した送信キュー制御を実装。
