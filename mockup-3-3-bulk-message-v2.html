3-3 一斉メッセージ・メール送信

加盟法人管理画面から対象会員を選択し、メッセージ / メール / 両方を一斉送信する機能

3つの送信方法の特徴とメリット・デメリットを比較します

A. アプリ内メッセージのみ

MessageThread + Push通知
+ アプリ内の「メッセージ」画面に残る（履歴参照可能）
+ Push通知でリアルタイムに届く
+ 既存の MessageThreadService + FCMService で実装可能
+ 未読管理あり（バッジ数表示）
- アプリ未インストール・通知OFF のユーザーに届かない
- HTML装飾不可（プレーンテキストのみ）
i FCMは500件/バッチで一斉送信可能

B. メールのみ

AWS SES 経由
+ アプリ未インストールでも届く
+ HTMLメールで装飾可能
+ 既存の EmailService (AWS SES) で実装可能
- 迷惑メールフォルダに入る可能性
- 開封確認は不確実
- アプリ内に履歴が残らない
i SES送信レート制限あり（バッチ処理で対応）

C. 両方送信

メッセージ + メール 同時送信
+ 到達率が最も高い（両チャネルでカバー）
+ アプリ内にも履歴が残り、メールでも確認可能
+ ユーザーの利用スタイルに関わらずリーチ
- 同じ内容が2回届く = うるさいと感じる可能性
- 送信処理が重い（MessageQueue でバッチ化推奨）
i ユーザー側でメール配信のオプトアウト設定を提供
推奨: 送信時に選択可能
項目 A. メッセージのみ B. メールのみ C. 両方送信
到達率 アプリ利用者のみ メール到達率依存 最も高い
即時性 Push通知で即時 メール確認タイミング依存 即時 + 後から確認
履歴参照 アプリ内で閲覧可能 メール受信箱のみ 両方で閲覧可能
装飾 プレーンテキスト HTML対応 メッセージ: テキスト / メール: HTML
既存基盤 MessageThreadService
FCMService		 EmailService
(AWS SES)		 両方を利用
オプトアウト Push通知OFF（OS設定） メール配信停止設定 チャネル別に設定可能
大量送信 FCM: 500件/バッチ SES: レート制限あり MessageQueue でバッチ処理
推奨 送信時に「メッセージ / メール / 両方」を選択できるUIを提供
→ 加盟法人が用途に応じて使い分けられる

実装概要

既存のMessageQueue・MessageThreadService・EmailService を活用

1. DBスキーマ追加
BulkMessageCampaign
（新規テーブル）		 id, companyId, companyUserId(送信者),
subject(メール件名), body(本文),
sendMode(1=メッセージ, 2=メール, 3=両方),
filterShopId?, filterPlanType?, filterContractStatus?, filterLastUsedDays?,
targetCount, sentCount, readCount,
status(pending/sending/completed),
createdAt, completedAt
AppUser
（フィールド追加）		 isBulkEmailEnabled Int @default(1) @db.TinyInt
— メール一斉配信のオプトアウト設定
2. API バックエンド
companyBulkSend 新規エンドポイント: 一斉送信を実行
1. フィルター条件で対象 AppUser を取得
2. BulkMessageCampaign レコード作成
3. MessageQueue に登録（バッチ処理用）
4. 即時にフロントへ受付完了を返す
companyBulkHistory 新規エンドポイント: 送信履歴一覧取得
BulkMessageCampaign のリスト返却
companyBulkTargetCount 新規エンドポイント: フィルター条件で対象者数を返す
リアルタイムでカウント表示に使用
3. バッチ処理（MessageQueue消化）
既存の MessageQueue モデルを利用。バッチ処理で allSent = 0 のキューを取得し、
lastSentAppUserId から再開してメッセージ/メール送信を実行。
メッセージ送信 MessageThreadService.sendMessageAsShop() を各ユーザーに実行
→ SHOP型スレッド作成 + FCM Push通知
メール送信 EmailProjectService.sendText() を各ユーザーに実行
→ isBulkEmailEnabled = 0 のユーザーはスキップ
進捗更新 BulkMessageCampaign.sentCount を更新
全員完了で status = completed, completedAt 設定
4. フロントエンド（Alliance）
会員一覧ページ 右上に「メッセージ・メール送信」ボタン追加
→ クリックで送信画面を展開 or 別ページに遷移
送信画面
